already in your repo

Required Files — No Changes Needed

These files already exist in the codebase and configure each platform automatically. You do not need to create or edit them to deploy.

render.yaml Tells Render how to build and run the backend — build command, start command, Node runtime, and all environment variable names. Render reads this file automatically when you connect the repo.
frontend/vercel.json Tells Vercel to redirect all routes to index.html so React Router works on page refresh. Without this, refreshing any page other than / returns a 404.
backend/prisma/schema.prisma The database schema. Render runs npx prisma migrate deploy on every deploy to apply any pending migrations to your Supabase database automatically.
backend/src/configs/env.ts Validates all environment variables at startup using Zod. If a required variable is missing or has an invalid value (like an empty string for a URL field), the server exits immediately with a clear error message.
01

Supabase

PostgreSQL database
01

Create a Supabase project

  1. Go to supabase.com → Sign up free with GitHub
  2. Click New Project
  3. Choose your organisation, give the project a name (e.g. marketplace-prod)
  4. Set a strong database password — save it, you'll need it
  5. Choose region: ap-northeast-1 (closest for most use cases)
  6. Click Create new project — takes ~2 minutes
02

Get the DATABASE_URL

Your backend uses a connection pooler URL, not the direct URL.

  1. In your Supabase project, click Connect (top bar)
  2. Click the Transaction tab under Connection String
  3. Copy the URL — it looks like:
postgresql://postgres.xxxx:[YOUR-PASSWORD]@aws-1-ap-northeast-1.pooler.supabase.com:5432/postgres
Important: Use the Transaction pooler URL, not the Direct or Session URL. The Transaction pooler handles connection limits on the free tier correctly.
03

Seed the production database

Migrations run automatically on Render deploy, but seed data (users, products, categories) must be run manually once.

# From your local machine, in the backend directory
cd backend
DATABASE_URL="your-supabase-url-here" npx prisma db seed

Upstash

Redis — queues & sessions
01

Create a Redis database

  1. Go to upstash.com → Sign up free
  2. Click Create Database
  3. Name it marketplace-redis
  4. Choose region: ap-northeast-1 (same as Supabase)
  5. Leave all other settings as default → click Create
02

Copy the credentials

In the Upstash dashboard, click your database → Details tab. You need these four values for Render:

Render VariableWhere to find it
REDIS_HOSTEndpoint field (e.g. happy-fox-12345.upstash.io)
REDIS_PORTPort field — usually 6379
REDIS_PASSWORDPassword field — click the eye icon to reveal
REDIS_TLSAlways true on Upstash — TLS is mandatory
Why TLS? Upstash requires encrypted connections. The backend's BullMQ job queue uses REDIS_TLS=true to enable it. Without this, queue workers silently fail to connect.
03

Render

Node.js backend
01

Create a Web Service

  1. Go to render.com → Sign up free with GitHub
  2. Click New → Web Service
  3. Connect your GitHub repository
  4. Render auto-detects render.yaml and pre-fills the build settings
  5. Do not change the build or start commands — they are set in render.yaml
02

Set environment variables

Go to the Environment tab and add every variable below. The variables marked manual are not in render.yaml — you must add them by hand.

VariableValueNote
DATABASE_URL postgresql://... manual — Supabase Transaction pooler URL
REDIS_HOST happy-fox-xxx.upstash.io manual — from Upstash dashboard
REDIS_PORT 6379 manual
REDIS_PASSWORD your-upstash-password manual — from Upstash dashboard
REDIS_TLS true Pre-set in render.yaml
JWT_PRIVATE_KEY -----BEGIN RSA PRIVATE KEY-----... manual — copy from local backend/.env
JWT_PUBLIC_KEY -----BEGIN PUBLIC KEY-----... manual — copy from local backend/.env
SESSION_SECRET any-32-char-random-string manual — generate: openssl rand -hex 32
APP_URL https://market-enterprise-app.onrender.com Your Render service URL
FRONTEND_URL https://your-app.vercel.app Update after Vercel deploy
CORS_ORIGINS https://your-app.vercel.app Update after Vercel deploy
JWT Keys: Copy the full multiline key from your local backend/.env. Each newline in the key must be a literal \n character, not an actual line break, when pasting into Render.
03

Deploy and verify

  1. Click Save Changes → Render starts deploying automatically
  2. Watch the Logs tab — first deploy takes 5–8 minutes
  3. Look for: [CORS] Allowed origins: [...] and Server started
  4. Confirm it's live by opening the health endpoint:
https://market-enterprise-app.onrender.com/api/v1/health

Expected response: {"success":true,"message":"OK"}

Free tier note: Render spins down after 15 minutes of inactivity. The first request after idle takes 30–60 seconds to wake the server. Use cron-job.org to ping the health endpoint every 10 minutes to prevent sleep.
04

Vercel

React frontend
01

Import the project

  1. Go to vercel.com → Sign up with GitHub
  2. Click Add New Project
  3. Import your GitHub repository
  4. Set Root Directory to frontend — this is required, the React app is not at the repo root
  5. Framework will auto-detect as Vite
  6. Leave Build Command and Output Directory as defaults
02

Set environment variables

On the same import screen, expand Environment Variables and add:

VariableValue
VITE_API_URL https://market-enterprise-app.onrender.com/api/v1
VITE_SOCKET_URL https://market-enterprise-app.onrender.com
VITE_STRIPE_PUBLISHABLE_KEY pk_test_your-stripe-key
VITE_APP_NAME Enterprise Marketplace
Stripe: Leave the default placeholder to run in Demo Mode (orders complete with one button, no card needed). Replace with real test keys from stripe.com to enable the full card payment UI.
03

Deploy and update CORS on Render

  1. Click Deploy — takes ~2 minutes
  2. Copy your Vercel URL (e.g. https://your-app.vercel.app)
  3. Go back to Render → Environment and update:
VariableUpdate to
FRONTEND_URLhttps://your-app.vercel.app
CORS_ORIGINShttps://your-app.vercel.app

Save → Render redeploys automatically. This step is required — without it the backend rejects all requests from Vercel with a CORS error.

Resend

Transactional email
01

Create an account and API key

  1. Go to resend.com → Sign up free (100 emails/day)
  2. Click API Keys in the left sidebar
  3. Click Create API Key → name it marketplace-prod
  4. Copy the key immediately — it starts with re_ and is shown only once
02

Add SMTP variables to Render

Go to Render → your service → Environment and add:

VariableValue
SMTP_HOSTsmtp.resend.com
SMTP_PORT465
SMTP_SECUREtrue
SMTP_USERresend
SMTP_PASSWORDre_xxxxxxxxxxxx
SMTP_FROM_EMAILonboarding@resend.dev
SMTP_FROM_NAMEEnterprise Marketplace
Custom domain: To send from your own domain (e.g. noreply@yourdomain.com) instead of onboarding@resend.dev, add your domain in Resend → Domains and add the DNS records it provides. Takes ~10 minutes to verify.
03

What triggers emails

No code changes needed — emails fire automatically once the SMTP variables are set:

  • Register → email verification link (expires 24h)
  • Forgot password → password reset link (expires 1h)
  • Place order → order confirmation with order number and total

Test by registering a new account with a real email address you own and checking the inbox.

Post-Deployment Checklist
Health endpoint responds — open https://market-enterprise-app.onrender.com/api/v1/health and confirm JSON response
Database seeded — ran npx prisma db seed with Supabase DATABASE_URL
CORS updated — FRONTEND_URL and CORS_ORIGINS on Render match your Vercel URL exactly (no trailing slash)
Login works — can log in with john.doe@example.com / Customer@123 on the live Vercel app
Email works — registered a new account and received the verification email in your inbox
Keep-alive configured — set up a cron job at cron-job.org to ping the health endpoint every 10 minutes so Render does not spin down
